3. Core Concepts Explained

3.1 React Components

What is a Component?

Components are reusable pieces of code that return JSX (JavaScript XML) to render UI elements. Think of them as building blocks for your website.

Example: 
function ProductCard({ product }) {
  return (
    <div className="product-card">
      <h3>{product.name}</h3>
      <p>₹{product.price}</p>
    </div>
  )
}

This component takes a product as input (props) and displays its name and price.

3.2 Props (Properties)

Props are how data flows from parent components to child components. They're like function parameters but for React components.

  • Parent Component: <ProductCard product={item} />
  • Child Component: Receives product as a prop

3.3 State Management

State is data that can change over time. When state changes, React automatically re-renders the component.

useState Hook Example: 
const [searchQuery, setSearchQuery] = useState('')

// searchQuery is the current value
// setSearchQuery is the function to update it
// '' is the initial value

3.4 Context API

Context API allows sharing data across multiple components without passing props through every level (prop drilling).

How it works:
  1. Create Context: Define what data to share
  2. Provider: Wrap components that need access to the data
  3. Consumer: Use useContext hook to access the data

In our project, ProductsContext provides product data to all components.

3.5 React Router

React Router enables navigation between different pages (routes) in a single-page application without page reloads.

Routes in our project:
  • / - Home page
  • /products - Products listing page
  • /product/:id - Individual product detail page

The :id is a dynamic parameter that changes based on which product is clicked.

3.6 useEffect Hook

useEffect runs side effects (like API calls, setting up timers) after the component renders.

Example from HeroSlider: 
useEffect(() => {
  const timer = setInterval(() => {
    setCurrentSlide((prev) => (prev + 1) % slides.length)
  }, 5000)
  return () => clearInterval(timer) // Cleanup
}, [slides.length])

This creates a timer that changes slides every 5 seconds. The cleanup function clears the timer when the component unmounts.

3.7 useMemo Hook

useMemo memoizes (caches) expensive calculations so they only run when dependencies change.

Example from Products page: 
const filteredProducts = useMemo(() => {
  let filtered = products
  // Apply filters...
  return filtered
}, [products, selectedCategory, searchQuery])

This only recalculates filtered products when products, category, or search query changes, not on every render.

5. Key Features Explained

5.1 Product Filtering

The Products page allows filtering by:

  • Category: Men, Women, Kids, Accessories
  • Rating: Filter by minimum star rating
  • Price Range: Set minimum and maximum price
  • Search: Search by product name or description
How filtering works:
  1. User selects filter options (stored in state)
  2. useMemo hook calculates filtered products
  3. Only products matching all criteria are displayed
  4. Component re-renders with filtered results

5.2 Search Functionality

The search bar in the Navbar allows users to search for products:

  • Search query is stored in state
  • On form submit, navigates to /products?search=query
  • Products page reads the search parameter from URL
  • Products are filtered to match the search query

5.3 Product Details Page

When a user clicks on a product:

  1. React Router navigates to /product/:id
  2. useParams hook extracts the product ID from URL
  3. getProductById function finds the product
  4. Product details are displayed with images, description, price
  5. Related products from the same category are shown

5.4 Image Gallery

The ProductDetails page features:

  • Main large image with zoom on hover
  • Thumbnail gallery below for multiple images
  • Clicking thumbnails changes the main image
  • Uses useState to track selected image index

5.5 Responsive Design

TailwindCSS utility classes make the website responsive:

  • md: prefix for medium screens and up
  • lg: prefix for large screens
  • Grid layouts adapt from 1 column (mobile) to 4 columns (desktop)
  • Navigation collapses on mobile devices

6. Data Flow

6.1 Products Data

Product data is stored in products.json and loaded through ProductsContext:

Data Flow:
  1. products.json contains all product information
  2. ProductsContext loads and provides this data
  3. ProductsProvider wraps the app, making data available everywhere
  4. Components use useProducts() hook to access products
  5. Components can filter, search, and display products

6.2 URL Parameters

URL parameters are used for:

  • ?category=Men - Filter by category
  • ?search=shirt - Search query
  • /product/5 - Product ID in the path

useSearchParams and useParams hooks read these values.

8. Common Questions & Answers

Q: What is React?

A: React is a JavaScript library for building user interfaces. It uses components (reusable pieces of code) to create interactive web pages. Instead of writing HTML directly, you write JSX (which looks like HTML but is actually JavaScript).

Q: What is JSX?

A: JSX (JavaScript XML) is a syntax extension that lets you write HTML-like code in JavaScript. React converts JSX into JavaScript that creates HTML elements.

// JSX
<div className="card">Hello</div>

// Converts to JavaScript
React.createElement('div', { className: 'card' }, 'Hello')

Q: What are Hooks?

A: Hooks are special functions that let you use React features in functional components:

  • useState - Manage component state
  • useEffect - Handle side effects
  • useContext - Access context data
  • useParams - Get URL parameters
  • useMemo - Memoize calculations

Q: What is a Single Page Application (SPA)?

A: An SPA loads once and then updates content dynamically without full page reloads. React Router handles navigation between "pages" (which are actually different components) without refreshing the browser.

Q: How does filtering work?

A: When a user selects a filter:

  1. The selection updates state (e.g., setSelectedCategory('Men'))
  2. useMemo recalculates which products match the filter
  3. React re-renders the component with filtered products
  4. Only matching products are displayed

Q: What is Context API used for?

A: Context API provides a way to share data (like product list) across many components without passing props through every level. It's like a global store that any component can access.

Q: How does React Router work?

A: React Router watches the URL and renders different components based on the path:

  • URL: / → Renders Home component
  • URL: /products → Renders Products component
  • URL: /product/5 → Renders ProductDetails with product ID 5

Q: What is the difference between state and props?

A:

  • Props: Data passed from parent to child (read-only)
  • State: Data managed within a component (can be changed)

Example: A product card receives product data as props, but the selected image index is state.
